# TDD Workflow

## 7 Phases

```
INIT → PLAN → RED → GREEN → REFACTOR → REVIEW → COMMIT
```

## Phase Details

| Phase | Content |
|-------|---------|
| INIT | Start cycle, create Cycle doc |
| PLAN | Define scope, create Test List |
| RED | Create failing test |
| GREEN | Minimal implementation |
| REFACTOR | Improve code quality |
| REVIEW | Quality check |
| COMMIT | Git commit |

## Absolute Rule

When you find an error, write a test first.
